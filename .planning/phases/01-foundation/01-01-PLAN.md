Create `.env.local.example` documenting the env var convention:
```bash
# .env.local.example — Copy to .env.local and fill in values. Never commit .env.local.
#
# SECURITY CONVENTION:
# - Server-only secrets (database, email): NO prefix. Access only inside createServerFn or Server Routes.
# - Public vars (site URL): VITE_PUBLIC_ prefix is acceptable ONLY for non-sensitive config.
# - NEVER prefix DATABASE_URL, RESEND_API_KEY, or NOTIFICATION_EMAIL with VITE_PUBLIC_.
#   Doing so bakes secrets into the client JS bundle visible to every visitor.

# Supabase — Transaction-mode pooler URL (port 6543). NOT the direct connection (port 5432).
# Get from: Supabase dashboard > Connect > Transaction mode
DATABASE_URL=postgresql://postgres.[project-ref]:[password]@aws-0-[region].pooler.supabase.com:6543/postgres

# Resend API key — for inquiry email notifications
# Get from: Resend dashboard > API Keys
RESEND_API_KEY=re_...

# The operator email address that receives inquiry notifications
NOTIFICATION_EMAIL=your-personal@email.com

# Optional: public site URL (safe to have VITE_PUBLIC_ prefix — not a secret)
VITE_PUBLIC_SITE_URL=https://wischosgift.com
```

Deploy to Vercel using the Vercel CLI. No `vercel.json` is required — Vercel auto-detects TanStack Start via the Nitro adapter in `vite.config.ts`.

```bash
# Install Vercel CLI if not present
npm install -g vercel

# Link project to Vercel (creates .vercel/ directory — add to .gitignore)
vercel link

# Deploy to production
vercel --prod
```

After the first deploy succeeds, set the required environment variables in Vercel for all three scopes (Development, Preview, Production):

```bash
vercel env add DATABASE_URL
# Enter the Supabase transaction-mode pooler URL (port 6543)
# Apply to: Development, Preview, Production

vercel env add RESEND_API_KEY
# Enter the Resend API key
# Apply to: Development, Preview, Production

vercel env add NOTIFICATION_EMAIL
# Enter the operator notification email
# Apply to: Development, Preview, Production
```

If the Supabase project or Resend account has not been created yet, set placeholder values (e.g. `DATABASE_URL=placeholder`) so the build does not fail — the actual values will be needed for Plan 02 (Supabase) and Phase 4 (Resend). The placeholder will cause DB operations to fail at runtime but will not prevent the build from succeeding.

After setting env vars, redeploy:
```bash
vercel --prod
```

Confirm the deployment URL returns HTTP 200 by visiting it in a browser or running:
```bash
curl -I $(vercel --prod 2>/dev/null | grep "https://") | head -5
```

Add `.vercel/` to .gitignore if not already present.
