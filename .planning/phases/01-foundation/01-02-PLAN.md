<objective>
Set up the Drizzle ORM database layer with a Supabase PostgreSQL connection using the transaction-mode pooler (port 6543). Define the inquiries table schema. Push the schema to Supabase using drizzle-kit. Confirm the table exists with the correct columns.

Purpose: The database layer must be correct from day one. Using the direct connection (port 5432) instead of the pooler is the most common and most damaging production failure pattern for Vercel + Supabase. This plan locks in the correct connection configuration before any feature code touches the database.

Drizzle client — CRITICAL: prepare: false is mandatory for Supabase transaction pool mode:
```typescript
// src/server/db.ts
import { drizzle } from 'drizzle-orm/postgres-js'
import postgres from 'postgres'
import * as schema from './schema'

// prepare: false REQUIRED — transaction pool mode does not support prepared statements
// max: 1 — each serverless invocation serves one request
const client = postgres(process.env.DATABASE_URL!, { prepare: false, max: 1 })
export const db = drizzle(client, { schema })
```

Drizzle schema — use generatedAlwaysAsIdentity() for PK, NOT serial (deprecated):
```typescript
import { pgTable, varchar, text, timestamp, integer } from 'drizzle-orm/pg-core'

export const inquiries = pgTable('inquiries', {
  id: integer().primaryKey().generatedAlwaysAsIdentity(),
  // ... fields
})
```

DATABASE_URL format: postgresql://postgres.[project-ref]:[password]@aws-0-[region].pooler.supabase.com:6543/postgres
(Port 6543 = transaction-mode pooler. Port 5432 = direct connection — DO NOT USE on Vercel.)

**src/server/db.ts** — Drizzle client singleton:
```typescript
import { drizzle } from 'drizzle-orm/postgres-js'
import postgres from 'postgres'
import * as schema from './schema'

if (!process.env.DATABASE_URL) {
  throw new Error('DATABASE_URL environment variable is not set')
}

// CRITICAL: prepare: false is mandatory for Supabase Supavisor transaction pool mode.
// Using prepare: true (the default) causes "prepared statement already exists" errors in production.
// CRITICAL: max: 1 limits each serverless function instance to one connection.
// See: https://orm.drizzle.team/docs/get-started/supabase-new
const client = postgres(process.env.DATABASE_URL, { prepare: false, max: 1 })

export const db = drizzle(client, { schema })
```

**src/server/schema.ts** — inquiries table definition matching FOUND-05 field list:

The required fields from REQUIREMENTS.md are: id, company_name, contact_name, role, email, phone, product_interest, quantity, timeline, message, created_at.

```typescript
import { pgTable, varchar, text, timestamp, integer } from 'drizzle-orm/pg-core'

export const inquiries = pgTable('inquiries', {
  id: integer().primaryKey().generatedAlwaysAsIdentity(),
  companyName: varchar('company_name', { length: 255 }).notNull(),
  contactName: varchar('contact_name', { length: 255 }).notNull(),
  role: varchar('role', { length: 255 }),
  email: varchar('email', { length: 255 }).notNull(),
  phone: varchar('phone', { length: 50 }),
  productInterest: text('product_interest'),
  quantity: varchar('quantity', { length: 100 }),
  timeline: varchar('timeline', { length: 255 }),
  message: text('message'),
  createdAt: timestamp('created_at').defaultNow().notNull(),
})

export type Inquiry = typeof inquiries.$inferSelect
export type NewInquiry = typeof inquiries.$inferInsert
```

Note: Do NOT use `drizzle-zod` to derive Zod schemas from this table definition. drizzle-zod is incompatible with Zod v4 (GitHub issue #4406). Standalone Zod schemas will be written independently in Phase 4.

The `src/server/` directory is a hard server boundary. These files must only ever be imported by:
- Server Functions (`createServerFn`)
- Server Routes (`createServerFileRoute`)
- drizzle-kit CLI (via drizzle.config.ts)

Never import from client components, route component functions, or any file without a server boundary.

Run drizzle-kit to push the schema to Supabase:

```bash

```

drizzle-kit will use `drizzle.config.ts` which points to `src/server/schema.ts`. It connects using `DATABASE_URL` from `.env.local`.

Note: drizzle-kit push uses the DATABASE_URL directly (which may be the pooler URL). For drizzle-kit itself, try using the direct connection URL (port 5432) in a separate `MIGRATION_DATABASE_URL` env var if needed. The runtime `db.ts` always uses the pooler URL.
