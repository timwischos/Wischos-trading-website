<task type="checkpoint:human-action" gate="blocking">
  <action>
    Run these steps IN ORDER before continuing to the production test.

    **Step 1: Ensure inquiries table exists in Supabase**

    Check: Supabase Dashboard → Table Editor — does the `inquiries` table appear?

    If YES → skip to Step 2.

    If NO → run in terminal (requires .env.local with DATABASE_URL set):
    ```bash
    npx drizzle-kit push
    ```
    When prompted, confirm the schema changes. Then verify the table appears in Supabase Dashboard → Table Editor.

    **Step 2: Apply RLS policies via Supabase SQL Editor**

    Go to: Supabase Dashboard → SQL Editor → New query

    Paste and run this SQL:
    ```sql
    -- Enable RLS on the inquiries table
    ALTER TABLE inquiries ENABLE ROW LEVEL SECURITY;

    -- Allow anonymous INSERT (anyone can submit an inquiry)
    CREATE POLICY "allow_anon_insert"
      ON inquiries
      FOR INSERT
      TO anon
      WITH CHECK (true);

    -- Block anonymous SELECT (inquiries are private)
    CREATE POLICY "block_anon_select"
      ON inquiries
      FOR SELECT
      TO anon
      USING (false);
    ```

    Verify: Supabase Dashboard → Authentication → Policies → inquiries table shows 2 policies.

    Note: If you see "policy already exists" errors, the policies are already applied — that is fine, continue.

    **Step 3: Set Vercel environment variables**

    Go to: Vercel Dashboard → wischos-trading-website project → Settings → Environment Variables

    Ensure these are set for the Production environment:
    - DATABASE_URL — Supabase transaction pooler URL (port 6543)
    - RESEND_API_KEY — from Resend Dashboard → API Keys
    - OPERATOR_EMAIL — your email address for receiving inquiry notifications

    **Step 4: Deploy to Vercel**

    Run in terminal:
    ```bash
    vercel --prod
    ```

    Wait for deployment to complete. Note the production URL.

    **Step 5: Verify Resend domain (if not already done)**

    Go to: Resend Dashboard → Domains — check if wischosgift.com is verified (green checkmark).

    If NOT verified: Add domain → follow DNS instructions → wait for verification. This may take minutes to hours.

    If your domain is not verified yet, the email notification will fail (but the DB insert will succeed). You can still complete the production test — the form submission will work and save to Supabase, you just may not receive the email until the domain is verified.
  </action>
  <resume-signal>Type "ready" when all steps are complete (or "partial: no domain" if domain verification is pending)</resume-signal>
</task>
