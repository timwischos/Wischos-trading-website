## Decisions Made

- **Loader typed as `any`**: TanStack Start 1.166.7 registers `ssr: true` in the `@tanstack/react-start` module augmentation. This causes `ValidateSerializableLifecycleResult` to use the SSR serialization path, which evaluates the loader return type as `never` when no `TLoaderFn` is explicitly resolved. The `as any` cast with an `eslint-disable` comment bypasses this framework type system limitation while preserving correct runtime behavior.
- **DialogTitle sr-only**: Radix Dialog emits a console warning if `DialogTitle` is missing from `DialogContent`. Added `<DialogTitle className="sr-only">{product.name} — full size image</DialogTitle>` to satisfy the ARIA requirement without disrupting the image-only lightbox visual.
- **CTA link uses `?product=` query param**: Consistent with Phase 2 decision — Phase 4 inquiry form will read this param to pre-fill the product field.

## Issues Encountered

- TanStack Start 1.166.7 SSR type constraint: When `ssr: true` is registered in `@tanstack/react-start`'s `Register` interface (done automatically by routeTree.gen.ts), the `ValidateSerializableLifecycleResult` type evaluates the loader return type as `never` during TypeScript compilation. This is a framework limitation — the code works correctly at runtime. The `as any` cast is the accepted workaround until the framework types are fixed in a future version.
